====================================================
APRIL 3, 2026
====================================================

1. REVENUE DISCREPANCY INVESTIGATION
   - Michael reported dashboard retail sales ~$4K higher than Shopify Analytics
   - Screenshots: Shopify showed $47K / 588 orders, Dashboard showed $50,975 / 578 orders
   - Dashboard had FEWER orders but MORE revenue — ruled out date bleed issue

2. ROOT CAUSE: SAMPLE-BASED REVENUE EXTRAPOLATION
   - /daily-metrics fetched only 250 orders, summed revenue, multiplied by (totalOrders / 250)
   - Scale factor of ~2.3x amplified any sample bias — high-value orders in the sample inflated total
   - /sales-summary was even worse: only 50 orders sampled, then AOV * orderCount

3. FIX: PAGINATED FULL ORDER FETCH
   - Added fetchAllOrdersPaginated() helper function
   - Uses cursor-based pagination (Link header) to fetch ALL orders, not a sample
   - For ~600 orders/day, this is 3 pages of 250 = 3 API calls per store
   - Replaced sampling in /daily-metrics and /sales-summary with full fetch
   - Revenue, units, and top products now computed from ALL orders — exact, not estimated

4. UNFULFILLED ENDPOINT — ALSO FIXED
   - /unfulfilled had same sampling issue (limit=250, then scaled units)
   - Now paginates all unfulfilled orders for exact unit counts and date breakdown

5. DATE-STAMPED CACHE KEYS
   - Old keys: "cache:daily-metrics", "cache:sales-summary" (static)
   - New keys: "cache:daily-metrics:2026-04-03" (includes date)
   - Cache auto-invalidates at midnight CT instead of potentially serving stale previous-day data

6. REMAINING ~$250 GAP INVESTIGATION
   - After full fetch fix, dashboard was still ~$250 higher than Shopify Analytics
   - Built temporary /test-refunds endpoint to diagnose
   - Researched Shopify Analytics "Total sales" formula vs API total_price field

7. THREE CAUSES OF SMALL DISCREPANCY IDENTIFIED
   a. Gift card PURCHASES excluded from Shopify Analytics (liability until spent)
      - li.gift_card === true on line items identifies gift card products
      - Payments MADE WITH gift cards still count as sales (no change needed)
   b. Cross-day refunds: Shopify Analytics deducts refunds processed today on older orders
      - Refund attributed to refund date, not original order date
      - Dashboard only sees today's orders, misses refunds on past orders
   c. total_price never updates after refunds (current_total_price does)
      - Tested: no partial refunds on today's orders, so no impact this day

8. FIX: GIFT CARD AND CROSS-DAY REFUND ADJUSTMENTS
   - Added fetchCrossDayRefunds() helper function
     - Fetches partially_refunded + refunded orders updated today
     - Sums refund amounts on orders NOT placed today
   - Updated computeMetrics in /daily-metrics:
     - Subtracts gift card line item revenue from sales
     - Subtracts cross-day refund amounts
     - Excludes gift-card-only orders from order count
   - Updated computeSales in /sales-summary: same adjustments
   - Applied to BOTH retail and wholesale stores
   - After fix: dashboard matched Shopify Analytics within a few dollars

9. VERIFIED DATA ACCURACY ACROSS ALL DASHBOARDS
   - Cross-checked all endpoints for consistency:
     - Daily Metrics sales = Sales Summary today's sales (MATCH)
     - Daily Metrics orders = Sales Summary today's orders (MATCH)
     - AOV = sales / orders (MATCH)
     - Fulfilled + Unfulfilled = totalOrders (MATCH)
     - Unfulfilled endpoint total = Fulfillment Dashboard unfulfilled (MATCH)
   - Confirmed frontends display raw API values with no client-side manipulation
   - Full audit table of all dashboards: exact vs estimated
